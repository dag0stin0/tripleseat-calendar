# PubKey Event Dashboard — Design System

---

## Design Context

### Users & Purpose

| | |
|---|---|
| **Who** | PubKey ops team + ownership (5-10 people) |
| **Where** | Laptop at the bar, phone in a cab, monitor in the office |
| **Job** | See what's happening this week. Filter by location, status, room. Get a pulse on event density and guest volume across NYC and DC. |
| **Emotional goal** | Confidence and clarity — a reliable command center, not a cluttered spreadsheet |

### Brand Personality

**Curated. Community-driven. Authentically cool.**

PubKey is a crypto/Bitcoin-focused social venue — not a tech startup, not a nightclub. It sits at the intersection of underground culture and professional community building. Events range from political fundraisers to comedy shows to trivia nights — eclectic but intentional. The vibe is *"we know what we're doing"* without being corporate.

### Anti-patterns

Do not do any of the following:
- Dark mode glow effects, neon accents
- Gradient text on headings or metrics
- Glassmorphism or blur-heavy decoration
- Crypto-bro aesthetic (blockchain visuals, coin imagery)
- Rounded cards with thick colored borders
- Bounce/elastic easing
- Icons above every heading

This should feel more like **Ramp** than Coinbase.

## Aesthetic Direction

**Apple-clean minimal** with data-forward density where it counts.

### Theme

Light mode. Clean white surfaces (`#ffffff`) on a soft warm-gray canvas (`#f8f8fa`). Surfaces separate through subtle 1px borders (`#e4e4ea`) and background shifts — not shadows or elevation.

### Typography

| Role | Font | Weight | Size | Notes |
|---|---|---|---|---|
| Body / labels | Space Grotesk | 400–600 | 11–13px | Geometric, modern, clean |
| Data / numbers | JetBrains Mono | 500 | 10–32px | Monospace for alignment and emphasis |
| Stat values | JetBrains Mono | 600 | 32px | `-1px` letter-spacing, orange |
| Section titles | JetBrains Mono | 600 | 11px | Uppercase, `1.5px` letter-spacing, tertiary color |
| Event names | Space Grotesk | 600 | 12px | `-0.1px` letter-spacing |
| Tags/badges | JetBrains Mono | 600 | 8–10px | Uppercase, tight tracking |

**Rules:**
- Never use weights outside 400/500/600 (we only load these)
- JetBrains Mono is for data, not prose
- No decorative type treatments (gradient, outline, shadows)

### Spacing

Base unit: **4px**. Common values: 6, 8, 10, 12, 14, 16, 20, 24, 28, 36, 48.

| Context | Value |
|---|---|
| Card internal padding | 8–10px (event cards), 14–24px (stat/room cards) |
| Grid gaps | 1px (seam style), 4–8px (loose elements) |
| Section spacing | 20–36px margin-bottom |
| Page horizontal | `clamp(16px, 3vw, 40px)` — fluid |
| Max content width | 1400px |

**Rules:**
- Grid seams use 1px gap with border-color background — creates hairline dividers without extra elements
- Generous section spacing, tight internal spacing
- No uniform padding everywhere — rhythm comes from contrast

### Borders & Radius

| Element | Radius |
|---|---|
| Large containers (calendar, stats row, table) | 10px |
| Buttons, selects | 6px |
| Tags, badges | 3–4px |

**Rules:**
- Containers use 1px solid `--border`
- No shadows for elevation — use background color shifts instead
- No thick accent borders on one side (except event card left-edge color coding, which is 2px)

### Motion

| Property | Value |
|---|---|
| Easing | `cubic-bezier(0.16, 1, 0.3, 1)` — smooth deceleration |
| Duration | 0.15–0.2s for interactions |
| Hover transforms | `translateY(-1px)` on cards, `scale(0.97)` on button press |

**Rules:**
- Only animate `transform` and `opacity` (GPU-accelerated)
- No bounce or elastic easing
- No entrance animations on data — it should feel instant
- Hover states are subtle, not theatrical

### Responsive Breakpoints

| Width | Behavior |
|---|---|
| > 1200px | 7-column calendar, full layout |
| ≤ 1200px | 4-column calendar |
| ≤ 768px | 2-column calendar, stacked stats, wrapped topbar |
| ≤ 480px | 1-column calendar |

## Design Principles

1. **Data first, decoration never.** Every pixel serves the person checking the calendar. If it doesn't inform a decision, remove it.

2. **Quiet confidence.** The design should feel assured and refined — like it was made by someone who doesn't need to prove anything. Subtle > loud.

3. **Scannable in 2 seconds.** A glance tells you: how many events this week, where they are, what's coming up. Visual hierarchy does the work so the user doesn't have to.

4. **Consistent restraint.** One accent color. One display font. One data font. Fewer tools, used with precision.

5. **Responsive to context.** Laptop at the bar, phone in a cab, monitor in the office. Adapts layout without losing information or clarity.
